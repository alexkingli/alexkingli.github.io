Flink Checkpoint

一. 基础概念

1.1 checkpoint

概念： 将某一个时间点所有的 Flink 算子的状态进行持久化
这些持久化状态叫做 轻量分布式快照
这个过程就是 checkpoint
目的：作业失败时能从最近的状态恢复，保证计算时数据一致性

1.2 有状态的计算

场景示例：
数据源：网络访问日志
需求：实时统计每个 url 的访问次数
状态：被“保存”起来的算子中的计算结果
